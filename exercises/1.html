# 题目

1、一个项目刚刚完成验收测试，仍有2%已识别到的问题未解决，尽管这些问题并非重要问题，但相关方不愿接受可交付成果，因此该项目无法进入下一阶段，若要避免这个问题，项目经理应该事先做什么?

  • A.在项目范围说明书中明确定义验收标准
  • B.在开发过程中执行更多质量控制程序，以防止在验收测试之前出现问题
  • C.定义更宽松的控制界限，以便所有可交付成果都能通过验收界限
  • D.让具有更高职权的项目发起人批准可交付成果，即使还有未解决的非关键问题

2、最新的项目状态报告显示进度绩效指数(SPI)为0.8,成本绩效指数(CPI)为1.3，它还表明项目团队自项目开始以来一直在加班工作。 项目经理应该怎么做?

  • A.增加更多资源，让项目回到正常的进度计划
  • B.提交变更请求，延长项目完成日期
  • C.开展团队绩效评估，必要时进一步建设团队
  • D.与团队共享状态，让项目回到正常的进度计划

3、—家公司开始一个IT项目，部署一个新系统。在执行阶段，项目经理意识到项目进度落后，并且没有额外的资源。项目经理应使用下列哪一项技术来解决进度差距?

  • A.资源平衡
  • B.假设情景分析
  • C.赶工
  • D.快速跟进

4、由于当初低估了技术难度，导致关键路径上的活动无法按时完成。项目经理意识到已无法获得额外的资源，而且关键路径上的工作无法并行。项目经理应该使用哪一项技术满足进度要求?

  • A.资源平衡
  • B.资源平滑
  • C.赶工
  • D.快速跟进

5、考虑到产品一直都能满足质量要求，团队成员逐渐降低了抽样率，并简化了测试流程，以降低成本。项目经理应该怎么做?

  • A.鼓励团队成本优化
  • B.提高质量测试标准
  • C.更新质量管理计划
  • D.执行定期质量审计

6、项目需要一名熟练的工程师来执行某个特定任务，但该工程师因为正承担其它项目的大量任务，无法抽出足够精力参与这个项目，项目经理下一步应该怎么做?

  • A、修改进度计划，以适应该资源
  • B、与职能经理谈判该资源的参与
  • C、培训项目团队去执行需要的活动
  • D、向发起人请求更多时间去寻找另一名资源执行该任务

7、两名项目团队成员一直对产品设计意见不一致。即使经过多次尝试，项目经理仍无法解决这个问题，项目现在落后于进度计划。项目经理应使用什么冲突解决技术来立即解决这个问题?

  • A.缓和/包容
  • B.妥协/调解
  • C.撤退/回避
  • D.强迫/命令

8、—位新团队成员加入项目团队。他来自一个崇尚自由的组织，由于习惯，团队的各种会议他经常迟到。当问到为什么迟到，该团队成员回应说他正忙于处理可交付成果，他认为应该优先处理可交付成果。项目经理应该怎么做?

  • A.暂停该团队成员的工作并解释这种行为是不允许的
  • B.再次与该团队成员会面，解释守时至关重要
  • C.忽略这种行为，因为这名团队成员正在确保工作不受影响
  • D.确保团队所有人都了解团队章程并遵守其中的规则

9、项目经理计划在一个月内部署一个全球性项目。但是，多名经理联系项目发起人，表示缺乏对项目可交付成果的认识。规划期间，项目经理应完成下列哪一项?

  • A.开展项目影响分析、安排进度活动并采取行动
  • B.审查项目章程澄清范围
  • C.安排每周风险审查会议
  • D.识别项目相关方，并将其包含沟通管理计划中

10、一个项目团队正在不同时区工作，团队成员之间经常出现误解，尽管项目经理试图解决这个问题，但这个问题一直没有得到解决。若要改善这种情况，项目经理应该怎么做?

  • A、在团队会议上解决这个问题
  • B、与每位团队成员分别解决这个问题
  • C、修订沟通管理计划
  • D、审查相关方的需求

11、一个合并跨国公司计费系统项目即将开始，项目经理识别到有三位经理对这项变化有抵触情绪并在项目需求调研过程中不断提出反对意见，项目经理下一步应该怎么做?

  • A.将他们关注的问题升级上报给项目发起人
  • B.发起项目变更请求以满足他们的期望
  • C.运用人际关系技能，管理他们关注的问题
  • D.通过管理相关方的需求来控制相关方参与

12、项目规划阶段，一位相关方担心新的项目可能影响他负责的业务，项目经理识别出虽然项目对这位相关方的影响可以忽略，但他有权叫停项目。项目经理应该怎么做?

  • A、与这位相关方保持高频率沟通，以确保他充分参与项目
  • B、尽可能满足这位相关方的需求以消除他的担心
  • C、定期将项目的进展汇报给这位相关方
  • D、保持对这位相关方的关注，避免他对项目产生消极影响

13、在敏捷团队中，团队成员能够及时了解彼此的工作进展和各自的困难，对于提升项目绩效至关重要，可以通过以下哪种方式解决这一问题?

  • A.冲刺规划会
  • B.冲刺评审会
  • C.冲刺回顾会
  • D.每日站会

14、敏捷团队中，大家对同一个任务提出不同的解决方案，应该如何形成决策?

  • A.由项目发起人决定
  • B.团队成员举手投票决定
  • C.由敏捷专家(Scrum Master)决定
  • D.由产品负责人(Product Owner)决定

15、销售部门提出一项有利于促进销售的新功能需求，敏捷团队应该如何应对?

  • A.因为冲刺规划已经确定，拒绝这项需求
  • B.马上投入开发这项新功能
  • C.如果团队有余力就接受并开发这项新功能
  • D.由产品负责人(PO）列入产

16、公司正在开发一款餐厅服务机器人，软件团队建议采取敏捷开发方法，硬件团队认为应该坚持瀑布开发模式，那么应该怎么解决这个矛盾?

  • A.说服硬件团队改用敏捷开发以保持和软件开发一致
  • B.说服软件团队采用瀑布开发以保持和硬件开发一致
  • C.让软件团队采取敏捷，硬件团队采用瀑布，保持混合开发模式
  • D.上报给项目发起人，由项目发起人决定采取哪种开发方式

17、敏捷专家Scrum Master并不要求团队成员向他汇报工作，反而主动热心为团队成员解决他们的具体困难，以及提供无微不至的支持和帮助，以便于团队成员可以专注于自己的业务。这属于以下哪种领导力风格?

  • A.放任式领导
  • B.仆人式领导
  • C.变革式领导
  • D.交易式领导

18、—个项目正在开发将为组织提供竞争优势的技术。项目经理了解到，很多人认为一项刚刚出现的新技术使原先计划推出的技术彻底过时。项目经理应该怎么做?

  • A.忽略该技术发展，并关注项目的原始范围
  • B.建议使用新技术来跟上最新的技术发展
  • C.执行成本效益分析，以确定升级是否合理
  • D.完成影响评估并为项目建议最佳可选方案

19、一位经验丰富的项目经理加入一家新公司，并参与一个大型的全球项目，该项目已经运行多年，项目经理希望了解其预期结果，项目经理应该参考哪一份文件?

  • A.商业论证
  • B.项目管理计划
  • C.需求管理计划
  • D.效益管理计划

20、一位团队成员发现一项新法规可能影响一个异地项目的交付日期，项目经理下—步应该怎么做?

  • A.要求项目发起人接收该风险
  • B.在进度计划中添加额外时间
  • C.更新风险登记册进行风险分析
  • D.请主要相关方将该问题提

21、在风险管理会议期间，一名团队成员识别到一个关键供应商将可能停业。失去这个供应商将无法交付最终产品。团队审查并更新必要的关键组件规格，用于评估替代供应商是否符合资格。这使用的是什么风险应对策略?

  • A.回避
  • B.转移
  • C.减轻
  • D.接受

22、项目A已经交付给客户A，而且开始开展收尾活动。项目团队将会转到客户B的项目上工作。销售经理要求项目经理提供项目A的售后支持。项目经理应执行下列哪一项?

  • A.将项目团队转移到项目B上，开始新的启动活动.
  • B.建议销售经理获得新的资源
  • C.确保项目A的特定知识已获收集，并移交组织
  • D.拒绝销售经理的请求，并查阅合同.

23、一个供应商提供的材料不符合项目要求，该供应商称该要求不属于合同的一部分。若要避免这个问题，项目经理应该事先做什么?

  • A.审查采购工作说明书(Sow)
  • A.进行内部检查
  • B.执行风险识别活动
  • C.对供应商的场所进行检查

24、因突发的风险导致项目的成本将远远超出预算，发起人要求项目经理停止项目，项目经理应该怎么做?

  • A.准备变更请求，更新项目范围
  • B.记录项目情况，准备最终报告
  • C.寻找新的发起人，更新项目章程
  • D.与客户洽谈，继续完成剩余任务

25、若要产生项目的初步概要定义，供项目业主和参与者规划和衡量项目是否成功，下列哪一项是其中必要过程?

  • A.制定项目工作说明书
  • B.制定项目章程
  • C.制定WBS
  • D.收集需求
